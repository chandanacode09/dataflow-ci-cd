# Terraform Configuration for test-pipeline Triggers

This directory contains Terraform configuration to create Cloud Build triggers
for the test-pipeline pipeline.

## Triggers Created

- **CI Trigger** (`test-pipeline-ci`): Runs tests on Pull Requests
- **CD Trigger** (`test-pipeline-cd`): Deploys to Dataflow on push to main/master

## Automatic Apply

The CI/CD Agent automatically applies this Terraform configuration when creating
the pipeline. No manual steps required.

## Manual Operations

If you need to manually manage the triggers:

### Prerequisites

1. Install Terraform (>= 1.0)
2. Authenticate to GCP:
   ```bash
   gcloud auth application-default login
   ```

### Initialize Terraform

```bash
cd terraform
terraform init
```

### Review Changes

```bash
terraform plan
```

### Apply Configuration

```bash
terraform apply
```

### Destroy Triggers

To delete the triggers:

```bash
terraform destroy
```
